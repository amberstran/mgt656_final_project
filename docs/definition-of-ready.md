# Definition of Ready (DoR)

A user story is considered **ready for sprint planning** if it meets the following criteria:

### 1. Clear Description
- The story clearly describes the user, feature, and benefit.
- Uses the standard **As a [user], I want [feature], so that [benefit]** format.

### 2. Acceptance Criteria Defined
- Minimum of 3 acceptance criteria are defined.
- Criteria follow **Given-When-Then** format.
- Criteria are **testable** and unambiguous.

### 3. Story Points Estimated
- Effort estimated using the Fibonacci sequence (1, 2, 3, 5, 8, 13).

### 4. Dependencies Identified
- Any dependencies on other stories or tasks are documented.

### 5. Priority Assigned
- Priority is assigned: High, Medium, or Low.

### 6. Sprint Assignment
- Sprint iteration is defined if known (optional if part of backlog grooming).

### 7. No Outstanding Questions
- All major questions or unknowns clarified.
- Story is **negotiable**, but core functionality is clear.
