# Definition of Done (DoD)

A user story is considered **done** when it meets the following criteria:

### 1. Implementation Complete
- All tasks for the story have been implemented.
- Code adheres to project standards and conventions.

### 2. Code Reviewed
- Pull request has been reviewed and approved by at least one teammate.

### 3. Acceptance Criteria Met
- All acceptance criteria are satisfied.
- Behavior verified against Given-When-Then scenarios.

### 4. Testing
- Unit tests, integration tests, or manual tests have passed.
- No critical or blocking bugs remain.

### 5. Documentation Updated
- Relevant documentation (README, API docs, wiki) updated if needed.

### 6. Deployed / Demoable
- Code deployed to staging or test environment for verification.
- Story can be demonstrated to stakeholders.

### 7. No Outstanding Issues
- Any related issues, tasks, or blockers resolved.
